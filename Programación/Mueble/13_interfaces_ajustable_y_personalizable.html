1.3.- Interfaces Ajustable y Personalizable

IO-Images (Licencia Pixabay)

Algunas subclases de la clase Mueble tendrán que implementar una o varias de las interfaces que se describen a continuación:

1.- Interfaz Ajustable

La interfaz Ajustable será implementada por cualquier mueble que se pueda poner en varias posiciones. Para proporcionar esta funcionalidad habrá que escribir los siguientes métodos:

  • método obtenerPosicion(), que permite obtener la posición actual del mueble ajustable.
  • método subirPosicion(), que permitirá subir una posición del mueble ajustable y devolverá un int que indicará la nueva posición. Este método deberá lanzar la excepción IllegalStateException, las circunstancias en las que se lanzará dicha excepción se detallarán en las clases que implementan esta interfaz.
  • método bajarPosicion(), que permitirá bajar una posición del mueble ajustable y devolverá un int que indicará la nueva posición. Este método deberá lanzar la excepción IllegalStateException, las circunstancias en las que se lanzará dicha excepción también se detallarán en las clases que implementan esta interfaz.

2. Interfaz Personalizable

OpenClipart-Vectors (Licencia Pixabay)

La interfaz Personalizable será implementada por todas aquellas clases cuyas instancias dispongan de la capacidad de ser "personalizadas", como por ejemplo un Armario. Los métodos que obliga a implementar esta interfaz son:

  • método obtenerModulos(), que devolverá una cadena con los módulos que se han añadido al mueble personalizable.
  • método anyadirModulo(Modulo modulo), que añadirá al mueble personalizable un elemento de tipo Modulo y no devolverá nada. Este método deberá lanzar las excepciones IllegalStateException y NullPointerException, las circunstancias en las que se lanzarán dichas excepciones se detallarán en las clases que implementan esta interfaz.
  • método extraerModulo(), que extraerá del mueble personalizable un elemento de tipo Modulo y lo devolverá. Este método deberá lanzar la excepción IllegalStateException, las circunstancias en las que se lanzará dicha excepción también se detallarán en las clases que implementan esta interfaz.

Las clases que incorporen estas interfaces tendrán que implementar estos métodos "a su modo", dependiendo del comportamiento y las características de cada clase específica. Es posible que cada clase tenga que implementarlos de una manera diferente (polimorfismo) o no.
