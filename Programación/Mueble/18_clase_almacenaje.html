1.8.- Clase Almacenaje

Nice2PleaseU(Licencia Pixabay)

La clase Almacenaje será una heredera directa de la clase Mueble. Representa el concepto abstracto de un mueble de tipo almacenaje, es decir, que sirve para guardar y almacenar libros, ropa, etc. Los objetos de esta clase estarán caracterizados por las siguientes propiedades:

  • todas aquellas que herede por el hecho de ser una subclase de Mueble;
  • anchura del mueble de almacenaje, en cm. (real);
  • altura del mueble de almacenaje, en cm. (real);
  • número máximo de módulos que se le pueden añadir al mueble de almacenaje (entero). Este atributo no se podrá cambiar una vez creado el objeto.
  • array de elementos de tipo Modulo.
  • número de módulos añadidos (entero). Este atributo servirá como contador de elementos añadidos al array. 

Además de los atributos anteriores, esta clase también dispondrá de un par de atributos estáticos constantes públicos que servirán para establecer el rango del número de módulos permitidos:

  • número mínimo de módulos del mueble de almacenaje (MIN_MODULOS): 1;
  • número máximo de módulos del mueble de almacenaje (MAX_MODULOS): 20.

La clase dispondrá de un único constructor que recibirá todos los parámetros que recibe un constructor de la clase Mueble más los atributos de anchura, altura y número máximo de módulos que se le pueden añadir al mueble. Este constructor deberá hacer uso del constructor de su clase padre para evitar cualquier redundancia de código. Si el número máximo de módulos no se encuentra dentro del rango permitido, se lanzará una excepción de tipo IllegalArgumentException con el mensaje de error "ERROR: No se puede crear el mueble de Almacenaje. El número de módulos no está en el rango permitido: XXX", donde XXX es el número máximo de módulos no válido. Una vez comprobado este atributo, se pasará a actualizar todos los atributos específicos de esta clase. Además, deberá crearse el array, reservando un espacio para almacenar la cantidad de módulos máxima pasado como parámetro. Puesto que el número mínimo de módulos es 1, habrá que añadir el primer módulo al array, siendo este primer módulo del tipo "BALDA" (recuerda que este tipo estará entre los valores del enum definido en la clase Modulo).

El número de módulos añadidos al mueble de almacenaje puede variar a lo largo de su vida, como veremos más adelante. Por tanto, debe ser modificable. También debe ser accesible desde sus descendientes (atributo de tipo protected), para que algunos de éstos puedan llevar a cabo esa modificación. Cualquier otro atributo de objeto que tengamos, procuraremos siempre que sea privado para aislarlo lo máximo posible dentro de la clase y evitar posibles daños al objeto desde fuera.

Habrá que implementar también métodos getter para poder obtener el valor de los atributos específicos que incorpore esta clase: anchura, altura, número máximo de módulos y número de módulos añadidos.

Como en cualquier nueva clase, habrá que retocar una vez más el método toString donde habrá que ampliar la información que ofrece el método homónimo de su superclase, añadiendo la información específica de la clase Almacenaje para ofrecer el siguiente formato de salida:

Donde WWW,WW es la anchura del mueble, HHH,HH la altura, NNN los módulos máximos que puede tener un mueble de almacenaje concreto, AAA será el contenido del array donde se almacenan los módulos añadidos. 

Tipo:Estanteria Id:YYY Precio:ZZZ,ZZ Descripción:VVV Anchura:WWW,WW Altura:HHH,HH Módulos máximos:NNN Módulos añadidos:AAA

Además de todo lo anterior, hemos de tener en cuenta que la clase Almacenaje debe implementar la interfaz Personalizable, de manera que habrá que escribir el código correspondiente a los siguientes métodos:

  • obtenerModulos(), que devolverá en una cadena el contenido del array de módulos;
  • anyadirModulo(Modulo modulo), que añadirá al array un elemento de tipo Modulo. Ten en cuenta que deberá comprobarse antes si el parámetro modulo no es nulo. En caso de ser nulo, deberá lanzar una excepción del tipo NullPointerException con el mensaje de error "Error: el módulo a añadir no puede ser nulo". Además, deberá comprobarse si es posible añadir más módulos al mueble. En caso de no poder añadirse, el método deberá lanzar una excepción de tipo IllegalStateException con el mensaje de error "Error: no se puede añadir el módulo XXX. El número de módulos no puede superar el máximo permitido: YYY", siendo XXX el módulo que se ha intentado añadir e YYY el número máximo de módulos que puede tener el mueble de almacenaje concreto;
  • extraerModulo(), que extraerá del array un elemento de tipo Modulo, devolviéndolo. Este elemento deberá ser el módulo que está almacenado en la última posición del array de módulos. Igualmente, se deberá comprobar antes si es posible extraer el módulo. En caso de no poder extraerse, deberá lanzarse una excepción de tipo IllegalStateException con el mensaje "Error: no se puede quitar el módulo. El número de módulos no puede ser inferior a XXX", siendo XXX el número mínimo de módulos añadidos.

Esta clase no debe ser instanciable, pues es necesario definir con más precisión de qué tipo de mueble de almacenaje estamos hablando, es decir, si se trata de un Armario, Estanteria, etc., que son formas diferentes de representar un mueble de almacenaje. Eso significa que es posible que más adelante haya que "retocar" esos métodos dotándoles de alguna funcionalidad más específica. De hecho, dado que se trata de una clase abstracta, podría incluso no haberse implementado estos métodos y haber dejado el trabajo a sus clases "descendientes" que fueran instanciables, pero dado que la funcionalidad anterior habrá que hacerla siempre, es más productivo escribirla aquí una sola vez para evitar tener que escribirla en todas las clases que hereden de ésta.
