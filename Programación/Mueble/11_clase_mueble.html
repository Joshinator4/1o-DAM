1.1.- Clase Mueble

Msaeedsalem (Licencia Pixabay)

En la clase Mueble se definen los atributos mínimos para representar las características elementales de cualquier mueble que se pueda comercializar en la tienda de muebles online:

  • identificador del mueble (entero);
  • precio del mueble (número real entre 0,01 y 10000,00 euros);
  • descripción del mueble (texto con una descripción algo más detallada).

Tanto el identificador como la descripción del producto son atributos inmutables que una vez creado el objeto no podrán ser modificados. Se implementarán además como atributos privados a los cuales solamente se tendrá acceso desde esta clase. El precio, sin embargo, sí podrá ser modificable y además se podrá acceder a él desde clases que sean descendientes de ella (atributo protected). El identificador se calculará automáticamente de manera incremental cada vez que se cree un nuevo producto. Eso implicará que tendrás que incluir un atributo de clase para poder llevar a cabo ese control. El primer producto que se cree tendrá como identificador el valor 1.

Además de los atributos anteriores, esta clase también dispondrá de algunos atributos estáticos constantes públicos que servirán para establecer el rango de precios correcto:

  • precio mínimo de un mueble (MIN_PRECIO): 0,01 euros;
  • precio máximo de un mueble (MAX_PRECIO): 10000,00 euros.

Estas características serán comunes a todo mueble, ya sea un sillón, una silla, una mesa, un mueble de tipo almacenaje (estantería o Armario), etc. Sin embargo, no podrán crearse objetos de tipo Mueble por ser una clase demasiado "genérica" a la que le faltan aún algunas propiedades y métodos más específicos. Por eso será una clase abstracta. Ahora bien, aunque se trate de una clase abstracta, dispondrá de algunos métodos que habrá que implementar:

  • Métodos getter para cada uno de los atributos descritos anteriormente:
    • getid(), que devolverá el identificador del mueble;
    • getPrecio(), que devolverá el precio del mueble;
    • getDescripcion(), que  devolverá la descripción del mueble.
  • Método toString, que devolverá una cadena de caracteres que mostrará la representación como String de los atributos de un mueble con el siguiente formato:
Tipo:XXX Id:YYY Precio:ZZZ,ZZ Descripción:VVV

donde XXX representa el tipo de mueble (Silla, Sillon, Sofa, Mesa, Armario, etc.), YYY el identificador, ZZZ,ZZ el precio (con dos decimales) y VVV la descripción.

Esta clase dispondrá de un único constructor con dos parámetros (precio y descripción). Se lanzará una excepción del tipo IllegalArgumentException si el precio no se encuentra dentro del rango fijado por las constantes. El mensaje de error deberá ser "El precio no está en el rango permitido: XXX,XX", donde XXX,XX es el precio inválido, con dos decimales. El resto de parámetros no hay que comprobarlos.

Tanto la clase Mueble como todas sus subclases y las interfaces que éstas implementen deberán estar agrupadas en un paquete llamado mobiliario.

Implementación del método toString

El método toString de una clase se utiliza para devolver una representación textual del estado del objeto utilizando algunos de los atributos de la clase (no necesariamente todos). Se trata de un método que nos permite observar y analizar la situación actual del objeto mediante un String. La mayoría de las clases que se implementan en Java suelen disponer de una implementación "especializada" de este método que rescribe y/o amplía la funcionalidad del método toString heredado por su clase padre. Y eso será también lo que tendremos que hacer en esta tarea.

En el caso de esta clase, el resultado que va a mostrar el método toString no tiene especial complicación: las etiquetas "Tipo:", "Id:", "Precio:" y "Descripción:" junto con el valor de los atributos correspondientes a cada etiqueta para ese objeto:

Tipo:Mueble Id:YYY Precio:ZZZ,ZZ Descripción:VVV

El único valor que no se trata de un atributo del objeto es el "Tipo" (XXX), que podrás obtenerlo mediante el método getClass() que tiene cualquier objeto en Java. Este método devuelve un objeto de tipo Class, que a su vez dispone de un método llamado getSimpleName() que devuelve un String con el nombre de la clase.  El resto de valores (YYY, ZZZ,ZZ, VVV) serán los contenidos en sus atributos correspondientes.

Por ejemplo, si para un objeto de este tipo la clase es Mueble, el identificador es 1, la descripción "mueble de prueba" y el precio 58,50, el contenido del String devuelto debería ser algo así:

Tipo:Mueble      Id: 1 Precio:  58,50 Descripción:mueble de prueba

Retroalimentación

Dada la poca información que contendrá una instancia de esta clase, no hay muchas alternativas para manipular y representar esa información. Puedes optar por usar la concatenación de varios String, utilizar un StringBuilder o bien emplear el método estático String.format. Trabajando con esa última alternativa, la implementación del método te podría quedar así:

    @Override
    public String toString() {
        return String.format("tipo:%-11s id:%2d precio:%8.2f descripción:%-20s",
                this.getClass().getSimpleName(), 
                this.getId(), 
                this.getPrecio(),
                this.getDescripcion()); 
    }

Observa que se ha formateado para que el valor de cada atributo se muestre siempre mostrando un tamaño fijo y alineado a la izquierda, para facilitar la lectura cuando se listen varios objetos uno debajo de otro. Este esquema de espaciado es solo una sugerencia. Tú podrías no indicar ningún tamaño fijo, o uno diferente al que se muestra como ejemplo aquí, para mejorar la presentación en la salida de tus programas.

También es importante que incluyas la anotación @Override, pues se trata de un método que se ha heredado y que estás sobrescribiendo.

Recuerda que para que una clase sea declarada como abstracta o no instanciable necesita incorporar el modificador abstract.
